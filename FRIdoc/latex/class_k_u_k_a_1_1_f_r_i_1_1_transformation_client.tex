\doxysubsection{Detailed Description}
Abstract \mbox{\hyperlink{namespace_k_u_k_a_1_1_f_r_i}{FRI}} transformation client. 

A transformation client enables the user to send transformation matrices cyclically to the \mbox{\hyperlink{namespace_k_u_k_a}{KUKA}} Sunrise controller for manipulating the transformations of dynamic frames in the Sunrise scenegraph. Usually, these matrices will be provided by external sensors. ~\newline
 Custom transformation clients have to be derived from this class and need to implement the \mbox{\hyperlink{class_k_u_k_a_1_1_f_r_i_1_1_transformation_client_aa21b0759e9a7a22516752841bede1a3c}{provide()}} callback. This callback is called once by the client application whenever a new \mbox{\hyperlink{namespace_k_u_k_a_1_1_f_r_i}{FRI}} message arrives.

{\bfseries{This element is an undocumented internal feature. It is not intended to be used by applications as it might change or be removed in future versions.}}

{\footnotesize\ttfamily void Transformation\+Client\+::set\+Transformation (\begin{DoxyParamCaption}\item[{const char $\ast$}]{transformation\+ID,  }\item[{const double}]{transformation\+Matrix\mbox{[}3\mbox{]}\mbox{[}4\mbox{]},  }\item[{unsigned int}]{time\+Sec,  }\item[{unsigned int}]{time\+Nano\+Sec }\end{DoxyParamCaption})}



Provides a requested transformation matrix. 

{\bfseries{This element is an undocumented internal feature. It is not intended to be used by applications as it might change or be removed in future versions.}} ~\newline


A transformation matrix has 3x4 elements. It consists of a rotational matrix (3x3 elements) and a translational vector (3x1 elements). The complete transformation matrix has the following structure\+: ~\newline
 \mbox{[}Transformation(3x4)\mbox{]} = \mbox{[}Rotation(3x3) $\vert$ Translation(3x1) \mbox{]} 

All provided transformation matrices need a timestamp that corresponds to their time of acquisiton. This timestamp must be synchronized to the timestamp provided by the \mbox{\hyperlink{namespace_k_u_k_a}{KUKA}} Sunrise controller (see \mbox{\hyperlink{class_k_u_k_a_1_1_f_r_i_1_1_transformation_client_a98bd724fa7d308ceeb488881b986f2a0}{get\+Timestamp\+Sec()}}, \mbox{\hyperlink{class_k_u_k_a_1_1_f_r_i_1_1_transformation_client_a1a068f1dae75a8478a1e63a3ffd2a0f2}{get\+Timestamp\+Nano\+Sec()}}). 

If an update to the last transformation is not yet available when the \mbox{\hyperlink{class_k_u_k_a_1_1_f_r_i_1_1_transformation_client_aa21b0759e9a7a22516752841bede1a3c}{provide()}} callback is executed, the last transformation (including its timestamp) should be repeated until a new transformation is available.
