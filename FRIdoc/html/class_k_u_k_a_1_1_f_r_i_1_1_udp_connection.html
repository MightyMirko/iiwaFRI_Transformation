Receive a new FRI monitoring message from the KUKA Sunrise controller.

This method blocks until a new message arrives.

Parameters
bufferPointer to the allocated buffer that will hold the FRI message
maxSizeSize in bytes of the allocated buffer
Returns
Number of bytes received (0 when connection was terminated, negative in case of errors or receive timeout)

HAVE_SOCKLEN_T Yes - unbelievable: There are differences in standard calling parameters (types) to recvfrom Windows winsock, VxWorks and QNX use int newer Posix (most Linuxes) use socklen_t

check for timeout Because SO_RCVTIMEO is in Windows not correctly implemented, select is used for the receive time out. If a timeout greater than 0 is given, wait until the timeout is reached or a message was received. If t, abort the function with an error.

Replace FD_ZERO with memset, because bzero is not available for VxWorks User Space Aplications(RTPs). Therefore the macro FD_ZERO does not compile.
